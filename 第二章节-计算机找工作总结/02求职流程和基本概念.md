# 求职流程和基本概念

## 1.1 先搞清楚你在打什么仗

求职本质上是一个漏斗：

> 海投 → 简历筛选 → 笔试/OA → 技术面(多轮) → HR 面 → offer → 谈薪

每个环节都会刷掉一批人。你的目标不是"每一步都完美"，而是让自己在每一步的通过率都高于平均水平。

## 1.2 校招 vs 社招：规则完全不同

| 维度 | 校招 | 社招 |
|------|------|------|
| 考察重点 | 基础功底 + 潜力 | 项目经验 + 即战力 |
| 算法要求 | 高，几乎必考 | 中等，看公司 |
| 项目要求 | 有亮点即可 | 要能讲清技术深度 |
| 时间线 | 提前批(6-7月) → 正式批(8-10月) → 补录 | 全年，金三银四、金九银十高峰 |
| 竞争对手 | 同届毕业生 | 各级别在职工程师 |

**关键认知：** 校招是你最大的杠杆——门槛相对低、容错率高、可以"跳级"进大厂。一旦错过，社招的难度会指数级上升。

## 1.3 求职时间线规划

以校招为例，建议的时间线（往前推）：

- **大二暑假 ~ 大三上：** 打好基础，刷题 + 做项目
- **大三寒假：** 准备简历初版，开始研究目标公司
- **大三下(3-5月)：** 日常实习投递，积累面试经验
- **大三暑假(6-7月)：** 秋招提前批，这是最黄金的窗口
- **大四上(8-10月)：** 秋招正式批，全力冲刺
- **大四下(次年3-5月)：** 春招补录，最后机会

社招的时间线更灵活，但核心逻辑不变：**先准备好自己，再出去面试，不要边面边学。**
