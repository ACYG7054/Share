# 求职流程和基本概念

## 求职流程

> 海投 → 简历筛选 → 笔试 → 技术面(多轮) → HR 面 → offer → 谈薪

所以思考具体是在哪个环节挂掉的，针对性的去提升：

- **海投没人理**：打招呼语和简历有问题
- **笔试挂**：多刷题，或者邪修
- **面试挂**：八股不扎实、项目理解不深刻
- **HR 挂**：没背过 HR 面的八股，回答不够圆滑

## 求职基本概念

### 校招 vs 社招

- **校招**：面向应届生（毕业前一年开始），竞争相对小，门槛低，企业愿意培养，是应届生最大的红利
- **社招**：面向有工作经验的人，要求来了就能干活，一般要求至少 1-3 年经验
- **结论**：能走校招就走校招，校招身份非常宝贵，别浪费

### 秋招 vs 春招

- **秋招**（8-10 月）：岗位最多、HC（名额）最充足，是主战场
- **春招**（3-5 月）：秋招的补录，岗位少、竞争大，捡漏为主
- **日常实习**：全年都有，随时可投，尽早开始积累经验
- **结论**：秋招是决胜局，春招是补刀，实习越早越好

### 什么是 HC

HC = Head Count，就是招人名额。HC 满了就是没坑了，你再优秀也进不去。所以投递要趁早，越往后 HC 越少。

### 什么是八股

八股就是面试中常考的基础知识题，比如 Java 集合、多线程、Spring、MySQL 索引、Redis 缓存等。叫八股是因为这些题目问法和答法都比较固定，背就完了。

### 简历的核心逻辑

简历不是写你会什么，而是写你**做过什么、产出了什么**。

- **项目经历** > 技能清单
- 每段经历要有：**背景 → 你做了什么 → 结果/数据**
- 没有实习经历就写项目，项目也要包装出"解决了什么问题"
- 简历一页纸，别超过两页

### 投递渠道

| 渠道 | 适用场景 | 备注 |
|------|---------|------|
| **Boss 直聘** | 社招、日常实习 | 最主流，注意优化打招呼语 |
| **牛客网** | 校招、内推 | 校招信息最全 |
| **官网** | 大厂校招 | 部分大厂只接受官网投递 |
| **内推** | 所有场景 | 能内推就内推，简历直达，跳过筛选 |
| **实习僧** | 实习 | 中小厂实习机会多 |

### 面试轮次一般怎么安排

1. **一面（技术面）**：八股 + 项目拷打，考察基础是否扎实
2. **二面（技术面）**：深挖项目、场景题、系统设计，考察思维深度
3. **三面（总监面/交叉面）**：聊职业规划、技术视野，看你的潜力和匹配度
4. **HR 面**：薪资期望、稳定性、到岗时间，基本到这一步问题不大

> 不是所有公司都有这么多轮，小厂可能一面就给 offer，大厂可能还有加面。

### 什么是 OC、意向书、三方

- **OC（Offer Call）**：HR 电话通知你过了，口头 offer
- **意向书**：书面确认录用意向，有一定约束力但不是合同
- **三方协议**：学校、企业、学生三方签的就业协议，签了就有违约金，慎签
- **结论**：拿到 OC 别急着签三方，多比较几家再做决定
