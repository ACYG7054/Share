# 怎么写简历

## 简历的核心原则

- **一页纸：** 应届生一页足够，社招最多两页
- **量化一切：** 用数字说话，不要用"大量""显著"这类模糊词
- **STAR 法则：** Situation（背景）→ Task（任务）→ Action（你做了什么）→ Result（结果）
- **关键词匹配：** 仔细读 JD，确保简历里出现岗位要求的核心技术词

## 简历结构建议

```
基本信息（姓名、电话、邮箱、GitHub）
├── 教育背景
├── 专业技能（技术栈，分层列出：精通 / 熟悉 / 了解）
├── 项目经历（2-3 个，按 STAR 写）
├── 实习/工作经历（如有）
└── 其他（竞赛、开源、博客、证书）
```

## 简历的几个常见坑

- **"精通 xxx"：** 除非你真的能接住任何追问，否则用"熟悉"
- **大段文字描述项目：** 面试官平均看一份简历 10-15 秒，要用短句 + 数据
- **放照片/写年龄/政治面貌：** 海外公司完全不需要，国内大厂也不在意
