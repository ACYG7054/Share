# 怎么制作学习路线

学习路线不是千篇一律的，需要根据学历、学习能力来制定。

## 2.1 先做自我诊断

在制定路线之前，诚实回答三个问题：

1. **你的目标岗位是什么？** 后端、前端、全栈、算法、数据、测试、运维/SRE？
2. **你现在的水平在哪？** 科班基础是否扎实？刷过多少题？做过什么项目？
3. **你还有多少时间？** 3 个月、6 个月、还是 1 年以上？

不同起点、不同目标，路线差异巨大。下面以**后端开发**和**AI/算法方向**两条最热门的路线为例展开。

## 2.2 后端开发学习路线

### 第一阶段：地基（1-2 个月）

**编程语言：** Java 或 Go 二选一深入，Python 作为辅助。不要同时学三四门语言。

**计算机基础四件套**——面试反复考，必须过关：

- **数据结构与算法：** 数组、链表、树、图、排序、搜索、动态规划。不需要竞赛水平，但 LeetCode Medium 要能稳定做出来
- **操作系统：** 进程/线程、内存管理、文件系统、死锁。重点理解概念，不需要读内核源码
- **计算机网络：** TCP/IP 模型、HTTP/HTTPS、DNS、三次握手/四次挥手。这些是面试高频题
- **数据库：** SQL 基础、索引原理、事务 ACID、MVCC。MySQL 是标配，PostgreSQL 加分

### 第二阶段：技术栈（2-3 个月）

以 Java 后端为例：

- **框架：** Spring Boot → Spring Cloud（微服务）→ MyBatis/MyBatis-Plus
- **中间件：** Redis（缓存）、RabbitMQ/Kafka（消息队列）、Elasticsearch（搜索）
- **DevOps 基础：** Linux 常用命令、Docker、Git 工作流、CI/CD 概念
- **系统设计入门：** 分布式基础概念、CAP 理论、负载均衡、数据库分库分表

> 不要贪多：挑 2-3 个中间件深入学，能讲清原理 + 使用场景 + 踩过的坑，远比"我都用过"有价值。

### 第三阶段：刷题 + 项目（持续进行）

- **算法：** LeetCode 200-300 题打底，按专题刷（数组→链表→树→动态规划→图）
- **项目：** 至少一个能讲出亮点的项目（后面展开讲）

## 2.3 AI / 算法方向学习路线

### 第一阶段：数学 + 编程基础（1-2 个月）

- **数学三件套：** 线性代数、概率统计、微积分（重点：矩阵运算、概率分布、梯度下降的数学原理）
- **Python 生态：** NumPy、Pandas、Matplotlib 熟练使用
- **机器学习基础：** 从经典算法入手——线性回归、逻辑回归、决策树、SVM、KNN

推荐资源：吴恩达 Coursera 课程（经典永不过时）、李宏毅机器学习（中文友好）

### 第二阶段：深度学习 + 主流框架（2-3 个月）

- **深度学习核心：** CNN、RNN/LSTM、Transformer、Attention 机制
- **框架：** PyTorch（首选）、了解 TensorFlow
- **CV 或 NLP 选一个方向深入：**
  - CV：图像分类、目标检测(YOLO 系列)、图像分割
  - NLP：词向量、文本分类、序列标注、预训练语言模型

### 第三阶段：紧跟前沿，建立差异化（持续进行）

2024-2026 年 AI 岗位的现实：

- 大模型(LLM) 已经成为绕不开的话题——了解 GPT 系列、LLaMA、RAG、Agent 架构、Prompt Engineering
- AI 工程化能力越来越重要——模型部署、推理优化、LangChain/LangGraph、向量数据库
- 纯算法研究岗卷到极致，AI 应用开发/AI 工程师反而是蓝海

> 给转 AI 方向的后端同学：你的工程能力是巨大优势。很多 AI 团队缺的不是调模型的人，而是能把模型落地成产品的工程师。LangGraph + FastAPI 这类技术栈，后端转过来上手很快。

## 2.4 学习路线的通用原则

1. **以面试倒推学习：** 先看目标公司的面经，再决定学什么。不要闷头学一年发现方向偏了
2. **深度 > 广度：** 面试官问一个 Redis，能聊 30 分钟（数据结构、持久化、集群、实际踩坑），比你简历写 10 个技术词有用得多
3. **输出倒逼输入：** 写博客、做笔记、录视频，逼自己把知识讲清楚。能教会别人，你才是真懂了
4. **不要完美主义：** 学到 70% 就可以开始做项目和面试了，剩下的在实战中补
