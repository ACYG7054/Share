4月份预计做一个企业级别的实战项目，会在Github开源，这里做文档解读

## 项目简介

这是一款面向健身人群的**智能健身 Web 应用后端**，基于 **FastAPI + LangGraph** 构建多智能体协作架构。

核心设计：由调度智能体进行意图识别与任务分发，专业智能体分别负责各自领域：

- **知识问答助手**：检索生成健身知识
- **导航智能体**：推荐最近健身房
- **健身记录智能体**：记录用户饮食、训练计划和基本情况（计算基础热量，统计身高、体重、BMI 等）

支持流式输出提升交互体验。

---

## 核心功能

### 1. 多智能体协作

意图识别智能体负责任务分发，按需调用子智能体（LangGraph SubAgent）：

- **健身记录智能体**：记录用户的饮食和健身情况
- **导航智能体**：根据用户 IP 搜索最近健身房（Skills 封装调用工具的能力）
- **知识问答助手**：向量数据库多路召回（混合检索 + 联网搜索）

### 2. 多模态交互

- 支持语音输入
- 支持图片输出

### 3. 工作流优先

能流程化的地方流程化，用工作流编排即可，**不要太依赖 Agent**。

### 4. 评估体系

- 怎么制作测试集
- 怎么测试 RAG 和 Agent 的效果
- 每次发布新版本时，怎么评估 RAG 和 Agent 的效果

### 5. CI/CD 自动化

- 自动化部署
- 自动跑测试集

## 技术要求

| 要求 | 说明 |
|------|------|
| **并发处理** | 单机部署达到 1000 QPS+（必须） |
| **技术栈** | Python + LangGraph + MySQL + Redis + Milvus + Vue |
| **可观测性** | 系统搭建要有利于线上测试，LangGraph 出现 Bug 时能快速定位 |
| **限流** | 接口请求超出系统处理能力时需要限流，且流量可配置 |
| **代码解耦** | 代码开发一定要解耦，利于扩展；MultiAgent 可扩展，配置解耦 |
| **代码量** | 尽量控制在 10w 行以内 |
| **知识管理** | 多 Agent 系统中，Admin 用户需要额外的知识嵌入菜单（用于 RAG 检索） |
