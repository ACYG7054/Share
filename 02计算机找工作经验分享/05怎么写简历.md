# 怎么写简历

## 简历的核心原则

- **一页纸**：应届生一页足够
- **量化一切**：用数字说话，不要用"大量""显著"这类模糊词
- **STAR 法则**：Situation（背景）→ Task（任务）→ Action（你做了什么）→ Result（结果）
- **关键词匹配**：仔细读 JD，确保简历里出现岗位要求的核心技术词
- **PDF 格式**：永远用 PDF 投递，Word 发过去格式会乱

## 简历组成

```
基本信息
├── 教育经历
├── 专业技能
├── 工作经历（实习/正职）
├── 项目经历
├── 校园经历（可选）
└── 自我评价（可选）
```

下面逐一讲清楚每个模块该怎么写，**务必全部遵守，严格执行**。

---

## 1. 个人信息

### 简历左上角：头像

放自己的头像，要求：严肃、正装、美颜、微笑。不要愁眉苦脸，不要嘻嘻哈哈，如果不会笑，至少做到平淡、淡然。

Boss 直聘上也必须放真人头像，这是第一印象。很多人没头像是万万不可的，头像太严肃也不太好。尽量表现得阳光、正能量，计算机学子可能天生不爱笑，但如果你做到了，就 unique 了。

> 拍照推荐：去"海马体"拍专业证件照，或者用抖音 AI 证件照、夸克 AI 生成证件照。

### 简历上方居中：基本信息

姓名、手机号、邮箱、微信号、GitHub 地址、年龄、性别、政治面貌（党员/共青团员）、目标职位名称。

### 简历右上角：学校 Logo

调整大小与头像对齐。超级简历不能调头像大小的话，用其他 PDF 工具调整。

> **必须放，不能说学历低就不放了。自信！自信！自信！**

## 2. 教育经历

### 学校 Title

自己百度自己学校有什么 title，**必须加上**，学历低的可以选择性加多个：

1. 985
2. 211
3. 双一流
4. 双一流建设中
5. 省属高校
6. 双百工程
7. 全日制本科
8. ......

### 需要包含的信息

| 项目 | 示例 |
|------|------|
| 就读时间 | 2022.09 - 2026.06 |
| 专业/学历/院校/地点 | 软件工程、本科、计算机科学学院、北京 |
| 综合成绩 | 班级第一，GPA：4.5/5 |
| 年级 Rank | 1/365 |
| 荣誉奖项 | 学校和社会的各种荣誉证书 |
| 奖学金 | 如实填写 |
| 英语能力 | CET-4 534、CET-6 643 |
| 职务 | 班长、学生会主席（国企可能看，互联网可写可不写） |

### 个人博客

**必须有。没有博客的简历，统一认为是垃圾简历。** 必须养成记录的习惯。

贴一个语雀知识库分享链接或者其他博客链接都行，但一定要外网能访问。

- **博客首页**：记录你学习成长的经过。比如你是如何喜欢上计算机的、印象最深的一门课、一次失败的解决过程、一些人生价值观。不是写几十个字，而是写几千字，写故事，展现你的性格、三观、积极向上的态度。这些内容放博客里写，不要放简历里。
- **博客其他页**：写技术文档，展现你写文档的能力。

> 研究生同理，增加一个教育经历模块即可。

---

## 3. 专业技能

这块大同小异，去牛客、小红书 copy 别人的即可，但**必须把对应知识点弄懂**。

- 有层次地写出来，分清精通 / 熟悉 / 了解
- 太难的知识点可以先不写，或者简单写
- **没有实习的同学，专业技能放前面**，给 HR 快速筛选用；有实习的放后面

---

## 4. 工作经历

| 项目 | 示例 |
|------|------|
| 工作类型 | 实习 / 正职 |
| 公司名称 | （美团）北京三快在线科技有限公司（可以整点小 logo 放在公司名字前面） |
| 工作时间 | 2021.06 - 2022.09 |
| 岗位/部门/地点 | Java 后端开发、风控部门、北京 |

### 项目介绍

一两句话讲清楚，不要过长。实习往往做的点很窄，简洁即可。

> 示例：我负责美团优惠券系统中天天神券的功能开发，帮助营销团队利润上涨 20%。

### 主要工作内容

核心写法：**抛出背景 → 说明问题 → 用什么技术解决 → 取得什么成效**。

> 示例：新年大促期间，由于用户过多需要缓存商家信息以降低数据库压力，我利用内部框架添加 xxx 缓存，保证了 xxx QPS 下系统 0 故障。

每个项目写 **3 个点**就够了，只留核心部分。多了不好讲，你也记不住。

## 5. 项目经历

因为这部分可能大部分都是学习项目，技术栈堆得很多，所以重点写**技术亮点**。

每个项目 **3 到 4 个亮点**，写法同样遵循 STAR 法则：

> 示例：
> 1. 使用乐观锁的思想解决了高并发场景下的优惠券超发问题，并使用 Redis + Lua 脚本完成用户领券资格的校验，保证用户无法超额领券，并使用 RabbitMQ 将领取优惠券和创建订单异步解耦，提高了接口的响应速度。
> 2. ......
> 3. ......

## 6. 校园经历（可选）

可以用来凑一页，也可以表达个性化。**当你技术达标后，这部分肯定是要删除的。**

写和计算机相关的、能表达出**沟通、组织、协调、决策能力**的事情。有个性最好，如果没有或者不会写就别写了。

写法要求：起因经过结尾，写故事，至少几十个字。不会写就问 DeepSeek。

> 示例：具有团队协作精神。大三参加国家软件设计大赛时协调项目组内 5 名开发同学，并对编码遇到困难的同学提供帮助，最终顺利在 1 个月内完成项目的核心功能。

---

## 7. 自我评价（可选）

同样可以用来凑一页或表达个性化，**技术达标后可以删除**。

别写什么热爱学习、热爱上班、喜欢 996，就真实评价自己的优点和缺点。想不出的写点正能量座右铭。

> 示例：
> - **优点**：扎实编程基础，热爱技术创新，具备团队协作与问题解决能力
> - **缺点**：容易深入研究细节，忘记时间，导致开发周期延误
> - **座右铭**：我知道有些机会，一辈子只会出现一次，我希望能抓住

## 常见问题

### 格式

微软雅黑字体，想办法挤到 1 页，调整字体大小和行间距。

### 页数

- 没实习：一页，一个项目加轮子或两个业务项目
- 有实习：可以两页，实习经历放第一页

### 模板颜色

推荐红色、蓝色、紫色。**不要用黑色模板。**

### 技术亮点怎么写

一句话描述，**先总说再细说**。重点突出，只写核心亮点，多不代表强，精才代表强。

### 缺少业务思考

整个项目描述只堆叠技术栈、不思考业务意义，是不好的简历。

### 无关内容

和计算机无关的东西（象棋、化工等）不要放在技能或项目里。能表达个性的内容放在校园经历里，比如发论文、算法竞赛项目等。

### 文体风格

项目和实习部分要**严肃、官方**，不要讲故事。校园经历和自我评价部分可以写点个性化的东西。
