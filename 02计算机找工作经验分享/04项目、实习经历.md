# 项目、实习经历

## 什么样的项目能加分

面试官看项目，核心看三点：

1. **你做了什么：** 不是项目有多大，而是你负责的部分有没有技术含量
2. **你怎么做的：** 技术选型的理由、遇到的问题、怎么解决的
3. **你学到了什么：** 对技术的深入理解，不是只会 CRUD

**好项目的特征：**

- 有明确的技术难点可以聊（高并发、分布式、性能优化等）
- 有自己的思考在里面（为什么选 A 不选 B？有没有做过对比？）
- 最好能量化成果（QPS 提升了多少、响应时间降低了多少）

**避坑：**

- 烂大街的商城项目（除非你有独特的技术亮点）
- 只会说"我按照教程做的"
- 简历写了但问起来一问三不知

## 项目包装的正确姿势

同一个项目，包装方式不同，效果天差地别：

> **差的写法：** 使用 Spring Boot + Redis + MySQL 开发了一个博客系统
>
> **好的写法：** 基于 Spring Boot 构建博客系统，引入 Redis 缓存热门文章列表，将首页加载时间从 1.2s 降至 200ms；通过 MySQL 读写分离 + 索引优化，支撑日均 10 万 PV 的访问量

区别在于：**优秀的写法遵循star法则，有数据、有对比、有技术深度。**

## 项目亮点怎么快速提炼（Demo 方法）

拿到一个项目后，不要闷头看代码，用 AI 帮你快速搞清楚：

**第一步：理清全局**

把整个模块发给 AI，让它帮你梳理出：
- 这个模块有哪些子模块
- 涉及哪些数据库表
- 有哪些核心接口

**第二步：深入细节**

挑出你简历上要写的某个亮点，让 AI 具体解释这个亮点的实现逻辑，并给出demo实现：
- 入参出参是什么
- 核心处理流程是什么
- 用了什么技术方案、为什么这么选

> 这样做的好处是：你不仅能快速理解项目，面试时被问到细节也能讲清楚，而不是"我按教程做的"。

## 没有实习经历怎么办

- **打造深度的项目：** 进阶的项目深入学习，可以根据自己需求来购买，比如马丁的知识星球、JavaGuide、小傅哥、鱼皮等，项目经验积累的足够之后，就可以自己去找开源项目或者自己手敲
- **竞赛项目：** ACM、Kaggle、各类黑客松，有含金量的奖项很加分
- **自己造轮子：** 仿写一个简化版 RPC 框架、简化版 MQ、简化版容器——面试官最喜欢这类项目
- **开源贡献：** 哪怕是修一个 bug、写一段文档，都比没有强
