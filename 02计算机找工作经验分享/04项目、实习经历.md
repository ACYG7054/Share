# 项目、实习经历

## 什么样的项目能加分

- 项目本身的含金量比较高，C端并发项目强于B端业务项目
- 面试官对你项目亮点比较感兴趣，你负责的部分含金量较高（当然是包装，实习哪有啥含金量的活给你）

## 什么是好项目

- 有明确的技术难点可以聊，如高并发、分布式、性能优化等
- 有自己的思考在里面，为什么选 A 不选 B？有没有做过对比？
- 最好能量化成果，QPS 多少、响应时间多少
- 差异化项目，并非烂大街的商城项目、RPC、短链接、AI玩具项目等等，最好是自己vibe coding一个项目

## 项目包装的正确姿势

同一个项目，包装方式不同，效果天差地别：

> **差的写法：** 使用 Spring Boot + Redis + MySQL 开发了一个博客系统
>
> **好的写法：** 基于 Spring Boot 构建博客系统，引入 Redis 缓存热门文章列表，将首页加载时间从 1.2s 降至 200ms；通过 MySQL 读写分离 + 索引优化，支撑日均 10 万 PV 的访问量

优秀的写法遵循star法则，有数据、有对比、有技术深度。

## Github找的项目，怎么快速学习，写入简历

- 先学怎么vibe coding
- 让AI分析这个项目有哪些模块，有哪些数据库表，有哪些核心接口，模块间的关系，表之间的关系和接口的具体实现流程
- 让AI挑出几个亮点来，基于star法则描述该亮点，你挑5个写简历上
- 再针对这5个亮点去深入学习，理解每一个亮点的技术选型、具体代码实现、让AI写出该亮点的demo

## 没有实习经历怎么办

- **打造深度的项目：** 进阶的项目深入学习，可以根据自己需求来购买，比如马丁的知识星球、JavaGuide、小傅哥、鱼皮等，项目经验积累的足够之后，就可以自己去找开源项目或者自己手敲
- **竞赛项目：** ACM、Kaggle、各类黑客松，有含金量的奖项很加分
- **自己造轮子：** 仿写一个简化版 RPC 框架、简化版 MQ、简化版容器——面试官最喜欢这类项目
- **开源贡献：** 哪怕是修一个 bug、写一段文档，都比没有强
