# 进入公司后怎么快速适应

## 第一天：先做人，再做事

- **主动和同事打招呼**，简单自我介绍，记住周围同事的名字。第一印象很重要，阳光、主动、有礼貌，别闷头坐着不说话
- 加好 mentor 和组内同事的微信/飞书/钉钉，有问题知道找谁

---

## 第一周：把项目跑起来

### Git 一定要提前学会

不会 Git 到公司会非常尴尬。最少要会这些：

- `git clone / pull / push / checkout / merge`
- 分支管理：怎么从 develop 拉分支
- 冲突解决：基本的 merge conflict 处理

### 拉取代码，跑通项目

这是入职后的第一个任务。期间一定会遇到各种环境问题（依赖装不上、配置不对、端口冲突等），处理原则：

1. **先自己查 + 问 AI**：把报错信息扔给 Claude / ChatGPT，90% 的环境问题它都能解决
2. **确认自己思考过了，再问同事**：不要一遇到问题就去打断别人，但也不要卡太久，超过 30 分钟没解决就该问了
3. **问的时候带上你试过的方案**："我遇到了 xx 问题，我试了 A 和 B 都没解决，想请教一下"——这样同事会更愿意帮你

---

## 日常工作：处理领导下发的需求

### 接到需求后的标准流程

1. **理解需求**：需求是什么、为什么要做、影响范围多大。不清楚的地方**当场问**，不要自己猜
2. **看相关代码**：先看现有代码怎么写的，别上来就另起炉灶
3. **和 mentor 对方案**：复杂需求先说你打算怎么做，确认思路没问题再动手
4. **写代码 + 自测**：写完自己先测一遍，别提交一个跑都跑不起来的代码

### 几个容易踩的坑

- **不要闷头干**：做到一半发现方向不对，浪费的是自己的时间。有疑问就沟通
- **不要改不该改的代码**：只改需求相关的，别顺手"优化"别人的代码，容易出问题还得罪人

---

## 怎么快速成长

- **多看别人的代码**：看高级工程师怎么写代码
- **主动记录**：遇到的问题和解决方案记到博客或笔记里，一方面加深记忆，另一方面面试也能用
- **多问 mentor**：前面说过了，mentor 一般不会主动教你，但你问他，他会很热情地讲。主动的人才能学到更多，但是不要问没营养的
- **了解业务**：不要只埋头写代码，搞清楚你做的东西在整个产品里是干什么的，有业务认知的人才值钱
