# 不具备竞争力该怎么提升

学历不够、项目一般、没有实习，怎么在简历上体现亮点？以下几个方向，**每一个都能给简历加分**，挑适合自己的做。

---

## GitHub

GitHub 主页就是你的技术名片，别放个水项目就挂上去。

**怎么让 GitHub 好看：**

- **提交要多且持续**：保持绿色格子，说明你一直在写代码
- **有 Star 的项目**：哪怕只有几十个 Star，也比空白强。写一个解决实际问题的小工具、做一个有特色的开源项目
- **README 写好**：项目介绍、技术栈、架构图、使用方式，一个都不能少。README 质量直接反映你的工程素养
- **有价值的 PR**：给知名开源项目提过 PR（哪怕是修 typo、补文档），说明你有开源协作能力

---

## LeetCode

LeetCode 主页也是能加分的，尤其是目标薪资 15k 以上或者面大厂的同学。

**怎么让 LeetCode 好看：**

- **题量**：至少刷 200+ 题，能覆盖大部分面试高频题
- **竞赛分**：参加周赛，分数上 1800+ 就能写在简历上，说明你有算法能力
- **连续打卡**：保持刷题记录的连续性，和 GitHub 绿格子一个道理

**刷题策略：**

- 先把热题 HOT 100 刷完，这是面试最高频的题目集
- 每道题限时 30 分钟，做不出来就看答案，理解后第二天再做一遍

> 简历上写"LeetCode 刷题 300+，周赛 rating 1900+"，算法面试的底气就有了。

---

## 博客

**不要发自己的学习笔记！** 面试官看到"Java 基础笔记""Spring 学习记录"这种，直接跳过。

**应该写什么：**

- **优质文章整理**：去看高质量技术文章，读懂之后用自己的话重新整理，加上自己的理解和思考
- **前沿技术解读**：比如 LangGraph 怎么用、RAG 的最新方案、Agent 架构设计
- **踩坑实录**：在项目中遇到的真实问题、排查过程、解决方案，这种文章最有价值
- **源码分析**：挑一个中间件的核心功能，画流程图讲清楚原理，能写出这种文章说明你确实看过源码
- **方案对比**：比如"Redis 和 Memcached 在 xx 场景下的选型对比"，有数据有结论

> 博客的核心是展现你的**思考能力**，而不是搬运知识。

---

## 竞赛

**学校发的竞赛一律不参加**，含金量太低，简历上写了反而减分。

**值得参加的竞赛：**

- 大厂举办的编程竞赛（字节、阿里、腾讯等）
- Kaggle 数据科学竞赛
- 各类黑客松（Hackathon）
- ACM / ICPC（算法方向）

> 不需要拿第一，参与 + 有成果就能写在简历上。

---

## 云服务器部署

把项目部署到线上

**部署之后要做的事：**

- **压测**：用 JMeter / wrk 跑一下，知道你的系统能撑多少 QPS
- **限流**：超出处理能力时怎么办？加上限流策略（令牌桶、滑动窗口等）
- **安全防护**：SQL 注入、XSS、CSRF 这些基本防护要有
- **日志和监控**：接入日志系统，出了问题能快速定位
- **HTTPS + 域名**：花几十块买个域名，配上 SSL 证书，显得专业

> 简历上写"部署于云服务器，经压测支撑 xxx QPS"，比写"本地运行成功"强太多。

---

## 实习

路径就一条：**小厂 → 中厂 → 大厂**，一步一步打怪升级。

- 没有实习经历就先投小厂，降低预期，先拿到第一段实习
- 有了一段实习再冲中厂，有了中厂经历再冲大厂
- 每段实习都要有意识地积累亮点，为下一段跳板做准备

> 第一段实习是最难的，之后会越来越容易。

---

## 造轮子

自己手写一个简化版的中间件或框架，面试官最喜欢这类项目：

- 简化版 RPC 框架
- 简化版消息队列
- 简化版 Spring IOC 容器
- 简化版 ORM 框架

> 不需要写得多完善，核心是你理解了原理，并能讲清楚设计思路。

---

## 技术社区影响力

在技术社区保持活跃，也是一种竞争力：

- **掘金 / CSDN**：发高质量技术文章，积累阅读量和粉丝
- **小红书**：发学习路线、面试经验、踩坑总结，容易涨粉
- **Stack Overflow**：回答英文技术问题，锻炼英语 + 技术能力

> 简历上写"掘金 xx 阅读量""小红书 xx 粉丝"，是实打实的差异化。
